# Codex 全局记忆指引

## 核心原则
- 对话负责思考，文件负责结论。
- 项目工作优先使用 `task-memory` skill 创建的项目级记忆窗口。
- `~/.codex/memories/` 只保留给 Codex 自身的元记忆或试验内容，不作为项目任务的默认存放区。

## 项目工作时
1. 先找当前项目的 `memory/`。
2. 如果没有，就用 `task-memory` skill 初始化一个。
3. 按项目级记忆窗口维护任务状态，不要把结论只留在对话里。

## 文档与视觉产出
- 当任务涉及 `PPT`、`slides`、`deck`、`resume`、`简历`、`Word`、`docx`、`PDF`、`report`、`proposal`、`文档` 时，优先调用对应的 `pptx`、`docx`、`pdf` skill。
- 当输出需要明显提升审美、版式、主题统一性时，优先结合 `theme-factory`；如果需要更强的布局与视觉 refinement，再结合 `ui-ux-pro-max` 或 `frontend-design`。
- 不要默认生成“白底黑字 + 普通标题 + 大段 bullet”的平庸成品；先明确视觉方向，再决定配色、字体层级、留白和版式节奏。
- 对 PPT、简历、正式文档，优先追求专业、克制、统一，而不是花哨；避免默认蓝色、默认字体、过密排版、装饰元素堆叠。
- 交付前优先做一次视觉 QA：检查对齐、留白、层级、对比度、分页/换行、封面与正文的一致性，必要时渲染预览后再收尾。
